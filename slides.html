Conditional Andersen

Adding Path-Condition Awareness to Inclusion-Based Pointer Analysis

基于路径约束扩充的Andersen指针分析

一种可调的、轻量级的分支敏感技术

01. 动机：传统Andersen的痛点

传统Andersen分析是路径不敏感（Path-Insensitive）的，所有分支上的赋值被无条件合并。

// 示例代码 x = alloc o1; // pts(x) = {o1} q = alloc o2; // pts(q) = {o2} t = alloc o3; // pts(t) = {o3} if (cond A) p1 = q; // 只在 cond A 下执行 else p2 = t; // 只在 !cond A 下执行 p = phi(p1, p2); *p = x; // store 操作

传统Andersen求解结果

pts(p) = {o2, o3} → pts(o2) ⊇ pts(x) = {o1}, pts(o3) ⊇ pts(x) = {o1}

结论：o2 和 o3 是别名（都指向 o1）

但实际上，o2 只在 cond A 下被写入，o3 只在 !cond A 下被写入，它们永远不会同时指向 o1！

02. 核心 Idea：条件约束边

将路径条件作为约束边的守卫（Guard），每个 points-to 关系携带触发条件。

// 条件约束边表示 o2 --(alloc, ⊤)--> q // pts(q) = {(⊤, o2)} o3 --(alloc, ⊤)--> t // pts(t) = {(⊤, o3)} q --(copy, cond A)--> p1 // pts(p1) = {(cond A, o2)} t --(copy, !cond A)--> p2 // pts(p2) = {(!cond A, o3)} p1 --(copy, ⊤)--> p // cond A: o2 ∈ pts(p) p2 --(copy, ⊤)--> p // !cond A: o3 ∈ pts(p) // pts(p) = {(cond A, o2), (!cond A, o3)}
Store 派生边（核心创新）：
x --(store, ⊤)--> p ⇒
x --(copy, cond A)--> o2,    x --(copy, !cond A)--> o3
最终结果： pts(o2) = {(cond A, o1)}, pts(o3) = {(!cond A, o1)}
别名查询： SAT(cond A ∧ !cond A) = false → o2 和 o3 不是别名！

03. 核心 Insight

传统思路

路径敏感 = 为每条完整路径维护独立的 points-to 状态

问题：路径爆炸，不可扩展

例如：SPAS (Sui et al., 2011) 在 full-sparse SSA 上用 BDD 编码路径，开销巨大

我们的思路

路径条件感知 = 在 points-to 集合元素上标注条件，而非复制整个状态

优势：保持 Andersen 的流不敏感性，只增加条件维度

类似 ICON 在 Andersen 上加上下文敏感，我们在 Andersen 上加路径条件敏感

类比：Context-Sensitive 中的 K-Limiting

1-callsite-sensitive, 2-callsite-sensitive... → 控制上下文深度

我们提出：1-path-sensitive, 2-path-sensitive... → 控制路径条件嵌套深度

k=0 时退化为传统Andersen；k→∞ 时逼近全路径敏感

05. 与 SPAS (2011) 的本质区别

维度 SPAS (Sui et al.) 我们的工作
基础框架 流敏感 + 上下文敏感 (FSCS) on SSA 流不敏感 Andersen
路径表示 BDD编码完整CFG路径，绑定SSA版本 约束边上直接标注原子路径条件
分析对象 程序点（SSA版本）级别的points-to 变量级别的条件points-to集合
精度粒度 过程内完整路径敏感 可调k-limiting（类似k-CFA）
Store处理 χ/μ操作在SSA上处理MAY/MUST-DEF 派生条件copy边（对象→对象）
可扩展性 SPEC CPU上平均+18%时间开销 保持Andersen多项式复杂度
关键差异：SPAS是"为每条路径维护独立状态"；我们是"在单个状态内用条件区分来源"

06. Demo 详解：逐步推导

// 原始代码 x = alloc o1; q = alloc o2; t = alloc o3; if (cond A) p1 = q; else p2 = t; p = phi(p1, p2); *p = x;

Step-by-Step 条件约束求解

// Step 1: Allocations (Rule ALLOC) pts(x) = {(⊤, o1)} pts(q) = {(⊤, o2)} pts(t) = {(⊤, o3)} // Step 2: Copy with conditions (Rule COPY) if (cond A) p1 = q → pts(p1) ⊇ {(⊤∧cond A, o2)} = {(cond A, o2)} else p2 = t → pts(p2) ⊇ {(⊤∧!cond A, o3)} = {(!cond A, o3)} // Step 3: Phi merge (Rule PHI) p = φ(p1, p2) → pts(p) = {(cond A, o2), (!cond A, o3)} // Step 4: Store generates derived edges (Rule DERIVED-COPY) *p = x → ∀(π₁,o)∈pts(p), ∀(π₂,o')∈pts(x): o ⊇_{π₁∧π₂} o' → o2 ⊇_{cond A∧⊤} o1 → pts(o2) = {(cond A, o1)} → o3 ⊇_{!cond A∧⊤} o1 → pts(o3) = {(!cond A, o1)}

07. Inference Rules (1/2)：基础规则

[ALLOC] 分配
alloc(o, x) ⇒ pts(x) ⊇ {(⊤, o)}
分配创建无条件指向关系
[COPY] 条件复制
(π', o) ∈ pts(x)    copy(x, y, π) ⇒ pts(y) ⊇ {(π' ∧ π, o)}
如果x在π'下指向o，且y=x在π下成立，则y在π'∧π下指向o
[LOAD] 条件加载
(π₁, o) ∈ pts(x), (π₂, o') ∈ pts(o), load(x, y, π) ⇒ pts(y) ⊇ {(π₁ ∧ π₂ ∧ π, o')}
需要x指向o（π₁）、o指向o'（π₂）、加载在π下执行，三者同时满足
[STORE] 条件存储
(π₁, o) ∈ pts(x), (π₂, o') ∈ pts(y), store(x, y, π) ⇒ pts(o) ⊇ {(π₁ ∧ π₂ ∧ π, o')}
将y的值（在π₂下指向o'）写入x指向的对象o（在π₁下），在π下执行

08. Inference Rules (2/2)：派生边与简化

[DERIVED-COPY] Store分解（核心创新）
store(x, y, π), (π₁, o) ∈ pts(x), (π₂, o') ∈ pts(y) ⇒ o →(copy, π₁∧π₂∧π) o'
将间接赋值分解为对象到对象的条件copy边
[PHI] 条件合并
phi(x₁,...,xₙ, y, [π₁,...,πₙ]), (π', o) ∈ pts(xᵢ) ⇒ pts(y) ⊇ {(π' ∧ πᵢ, o)}
每个分支携带其路径条件与原始条件的合取
[CONTRADICTION] 矛盾消除
(π, o) ∈ pts(x), π ≡ ⊥ ⇒ 从 pts(x) 中移除 (π, o)
不可能路径上的指向关系被剪枝
[ALIAS] 条件别名判断
may-alias(o₁, o₂) ⟺ ∃π₁,π₂: (π₁,o₁)∈pts(p) ∧ (π₂,o₂)∈pts(p) ∧ SAT(π₁ ∧ π₂)
仅当存在可满足的共同条件时才判定为别名

09. K-Path-Sensitive：可调精度机制

类似上下文敏感中的 k-CFA，我们引入 k-Path-Sensitive 控制路径条件嵌套深度。

// 条件深度定义 depth(⊤) = 0 depth(c) = 1 // 原子条件 depth(¬π) = depth(π) depth(π₁ ∧ π₂) = depth(π₁) + depth(π₂) depth(π₁ ∨ π₂) = max(depth(π₁), depth(π₂)) // K-Limit 规则 (π, o) ∈ pts(x), depth(π) > k ⇒ (abstract(π, k), o) ∈ pts(x)
k值 行为 精度 复杂度
k = 0 所有条件被替换为 ⊤ 传统Andersen O(n³)
k = 1 只保留最近1个分支条件 单层分支敏感 O(n³ · m)
k = 2 保留嵌套if-else条件 嵌套分支敏感 O(n³ · m²)
k → ∞ 保留完整路径条件 逼近全路径敏感 可能指数级

n = 变量数, m = 分支条件数

10. 精度-开销谱系定位

核心洞察

我们在 Andersen 的可扩展性和路径敏感的精度之间开辟了一个新的连续调节空间。

类似 k-CFA 在上下文敏感中的角色，k-Path-Sensitive 允许用户根据客户端需求选择适当的精度级别。

11. 未来工作与待解决问题

理论层面

  • Soundness 证明：条件约束求解的单调性与不动点存在性
  • 复杂度分析：k-limiting 下条件空间的上界证明
  • 精度边界：k=0 ↔ 传统Andersen；k→∞ ↔ 全路径敏感

实现层面

  • 条件压缩：借鉴 BDD / SMT 压缩条件表达式
  • 选择性敏感：只对指针相关分支（如 null check）启用条件追踪
  • 与上下文敏感的联合：k-path + k-call-site 的交互效应

评估层面

  • 对比传统Andersen：别名精度提升多少？
  • 对比SPAS：性能优势多大？
  • 对下游客户端（bug检测、优化）的实际收益

12. 总结

Contribution 1：条件约束边机制

在 Andersen 包含约束上增加路径条件守卫，实现流不敏感但路径条件感知的指针分析。

Contribution 2：Store-to-Copy 分解

将间接赋值 *p = q 分解为带条件的对象到对象 copy 边，实现跨分支的精细别名判断。

Contribution 3：K-Path-Sensitive 粒度

提出类似 k-CFA 的可调精度机制，在 Andersen 可扩展性和路径敏感精度之间提供连续权衡。

Conditional Andersen = Andersen + Path-Condition Awareness + Tunable Precision
